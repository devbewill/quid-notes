# PROMPT: RECREATE QUID NOTES OPERATIVE ENGINE

**Goal:** Build a high-performance relational management application designed for lateral thinking, fluid documentation, and ruthless execution. The system must eliminate hierarchical folder structures in favor of a flat, metadata-driven architecture.

---

### 1. CORE PHILOSOPHY & ARCHITECTURE
- **Non-Hierarchical**: Information logic is relational, not nested. No folders.
- **Relational Integrity**: Tasks are first-class citizens derived from or linked to Notes.
- **Real-Time Synergy**: Every change must synchronize instantly across all clients without manual reloads.
- **Bi-Directional Context**: If a task is created from a note, it must always link back to its semantic origin.

### 2. TECHNICAL SPECIFICATIONS (RECOMENTED STACK)
- **Frontend**: Next.js (App Router), React, Tailwind CSS.
- **Backend/Database**: Convex (for real-time reactivity and serverless logic).
- **Animations**: Framer Motion (for fluid, high-end interaction feel).
- **Authentication**: Modern auth (Google/OTP) integrated with the database.

### 3. FUNCTIONAL REQUIREMENTS

#### A. The Semantic Engine (Notes)
- **Editor**: A lightning-fast Markdown editor.
- **Tagging**: A robust, color-coded tagging system. Tags are the primary navigation/filtering method.
- **Soft-Delete**: Deleted items enter a "trash" state for 30 days before permanent erasure.
- **Quick-Add**: A low-friction entry point to capture ideas instantly.

#### B. The Execution Engine (Tasks)
- **Status Lifecycle**: Tasks move through `idle` (To Do), `active` (In Progress), and `completed`.
- **In-Context Extraction**: Ability to highlight or select text within a note and mathematically transform it into an executable task.
- **Relational Binding**: Tasks must store a reference to the Note they originated from.

#### C. Multi-Surface Visualization
- **Feed (List/Table)**: A high-density view for rapid scanning and filtering by tags or status.
- **Brutal Kanban**: A minimal, borderless drag-and-drop board for workflow management.
- **Timeline**: A chronological progression of items, emphasizing when things were modified or are due.

#### D. Cybernetic Intelligence (AI)
- **Context-Aware Assistant**: An integrated AI chat that maintains a vector or relational context of the user's data.
- **Semantic Queries**: The user should be able to ask "What was I thinking about [Topic] last week?" or "Summarize my active tasks related to [Project]".

### 4. UI/UX GUIDELINES (STYLE AGNOSTIC)
- **Performant**: Zero-latency feel for all basic operations (typing, toggling).
- **Internationalization**: Full support for IT and EN locales.
- **Theme Support**: Seamless Light and Dark mode switching.
- **Visual Excellence**: Use high-end motion design, smooth transitions, and distinct micro-interactions to provide a "premium" software feel.
- **Creative Freedom**: You have full control over the visual theme, color palette, and typography. Aim for a design that reflects "Controlled Chaos" or "Technical Purity".

### 5. EXECUTION PLAN
1.  Initialize the project and the real-time database schema.
2.  Implement the flat metadata tagging system.
3.  Build the Markdown captured/note-taking engine.
4.  Develop the triple-view system (Feed, Kanban, Timeline).
5.  Integrate the AI orchestration layer.
6.  Polish with advanced animations and localization.
